## 1. インストールと起動

### ビルドして起動

```bash
git clone https://github.com/fukuyori/tbla.git
cd tbla
cargo build --release
./target/release/tbla              # 空のシートで起動
./target/release/tbla data.json    # 既存ファイルを開く
./target/release/tbla data.csv     # CSV を開く
./target/release/tbla data.xlsx    # Excel を開く
./target/release/tbla data.parquet # Parquet を開く
```

初回ビルドには 2〜3 分かかります（Polars のコンパイル）。以降はインクリメンタルで数秒。

### 推奨環境

| 項目 | 推奨 |
|------|------|
| OS | macOS / Linux / Windows |
| ターミナル | WezTerm, iTerm2, Alacritty, kitty, Windows Terminal |
| ターミナル幅 | 80 文字以上（120+ が快適） |
| 行高 | 24 行以上 |
| 配色 | 任意（tbla は RGB 直接指定なのでテーマ非依存） |

### 終了

`Ctrl+Q` または `ファイル → 終了`。

## 2. 画面構成

```
┌────────────────────────────────────────────────────┐
│ ファイル(F) 編集(E) 挿入(I) シート(S) データ(D) … │  ← メニューバー (F10 で開く)
├─────┬───────────┬───────────┬───────────┬─────────┤
│     │    A     │    B     │    C     │   D    │  ← 列ヘッダー
├─────┼───────────┼───────────┼───────────┼─────────┤
│  1  │   名前    │   売上    │   原価    │   利益  │
│  2  │   林檎    │   1500   │    800   │    700  │  ← データ（カーソル位置はハイライト）
│  3  │   蜜柑    │   2300   │   1200   │   1100  │
│ … │
├─────┴───────────┴───────────┴───────────┴─────────┤
│ Sheet1 | Sheet2 | + …                              │  ← タブバー（2 シート以上の時のみ）
├────────────────────────────────────────────────────┤
│  A1 | 選択 A1:B3 (2×3) | fx: 名前                 │  ← 数式バー
│  通常 | data.xlsx | F10:メニュー                   │  ← ステータスバー
└────────────────────────────────────────────────────┘
```

### 各エリアの意味

- **メニューバー**：F10 または Alt+F/E/I/S/D/O/H で開く
- **列ヘッダー (A B C…)**：`│` の上をマウスでドラッグすると列幅変更
- **行番号 (1 2 3…)**：1 始まり
- **数式バー**：現在のセル位置・選択範囲・セル内容（数式は `→` で評価結果も表示）
- **ステータスバー**：モード、ファイル名、ヒント、メッセージ

|------|------|
| `↑↓←→` | 1 マス移動 |
| `Tab` / `Shift+Tab` | 右 / 左 |
| `Enter` / `Shift+Enter` | 下 / 上（編集確定後の自動移動と統一） |
| `Home` / `End` | 行頭 / 行末（データのある最終列） |
| `Page Up` / `Page Down` | 1 ページ移動 |
| `Ctrl+Home` | A1 へ |
| `Ctrl+End` | データのある最終セル |
| `Ctrl+矢印` | データ境界までジャンプ（Excel の Ctrl+矢印 と同じ） |
| `Ctrl+G` | セル参照を入力してジャンプ（例: `B100`） |
| `Ctrl+PgUp` / `Ctrl+PgDn` | 前 / 次のシート |

### vim 風（オプション）

ホームポジションから動きたい人向け：

| キー | 動作 |
|------|------|
| `Ctrl+H/J/K/L` | 左 / 下 / 上 / 右 |
| `Ctrl+Shift+H/J/K/L` | 同上 + 選択範囲を拡張 |

### 選択

| 操作 | 動作 |
|------|------|
| `Shift+矢印` | 範囲拡張 |
| マウスで他セルへドラッグ | 範囲選択 |
| `Ctrl+A` | 全選択（データ範囲） |
| `Esc` | 選択解除 |

選択範囲があると **コピー・切取・クリア・書式適用** がその範囲に対して効きます。

| 任意の文字キー | 編集モード（上書き入力） |
| セルをダブルクリック | 同 F2 |

### 編集中の文字カーソル

| キー | 動作 |
|------|------|
| `←→` | 1 文字戻る/進む |
| `Home/End` または `Ctrl+A/E` | 行頭/行末 |

| `Delete` / `Ctrl+D` | カーソル位置を削除 |
| `Ctrl+K` | カーソルから行末まで削除 |

| キー | 動作 |
|------|------|

| `Tab` | 確定して右のセル |

| `Esc` | 編集を破棄 |

| 入力 | 解釈 |

| `TRUE` / `FALSE` | 真偽値 |
| `50%` | 数値 0.5 |

## 5. 数式と関数

### 数式の基本

`=` で始めると数式：

```
=A1 + B1
=SUM(C2:C100)
=IF(A1 > 0, "OK", "NG")
=VLOOKUP(D5, $A$1:$B$100, 2, FALSE)
```

### 参照の種類

| 記法 | 種類 | 行コピー時 | 列コピー時 |
|------|------|-----------|-----------|
| `A1` | 相対 | 変化する | 変化する |
| `$A1` | 列固定 | 変化する | 固定 |
| `A$1` | 行固定 | 固定 | 変化する |
| `$A$1` | 絶対 | 固定 | 固定 |

行・列の挿入/削除時に参照は自動調整されます。

### ポイントモード（マウス / 矢印でセル参照を挿入）

数式編集中、`=` `(` `,` `+` `-` `*` `/` `^` `&` `:` `<` `>` の直後で矢印キーを押すと、その方向のセルが参照として挿入されます：

```
1. A4 で「=」と入力
2. ↑ を押す → 数式バーが「=A3」に
3. 「+」を入力 → ポイントモード継続
4. ↑ を押す → 「=A3+A2」
5. Enter で確定
```

マウスでセルクリック / ドラッグでも参照を挿入できます。参照中のセルは**青**でハイライト、範囲指定中は**濃い青**。

### 集約関数の自動補完

`=SUM`、`=AVG`（または `=AVERAGE`）、`=MIN`、`=MAX`、`=COUNT`、`=COUNTA` を**引数なしで Enter** すると、隣接する数値範囲が自動で埋め込まれます：

```
A1=10, A2=20, A3=30 の状態で A4 に "=sum" → Enter
→ "=SUM(A1:A3)" に自動補完
```

ルール: 上 → 左の順で連続する数値ブロックを範囲化。テキストや空セルで止まる。

### 関数カテゴリ（全 79 関数）

#### 集計
`SUM`, `AVERAGE`/`AVG`, `COUNT`, `COUNTA`, `MIN`, `MAX`

#### 数学
`ABS`, `ROUND`, `ROUNDUP`, `ROUNDDOWN`, `CEILING`, `FLOOR`, `INT`, `MOD`, `POWER`, `SQRT`

#### 三角・角度
`SIN`, `COS`, `TAN`, `ASIN`, `ACOS`, `ATAN`, `ATAN2`, `RADIANS`, `DEGREES`

#### 対数・指数
`LN`, `LOG`, `LOG10`, `EXP`, `PI`

#### 統計
`STDEV`/`STDEV.S`, `VAR`/`VAR.S`, `MEDIAN`, `MODE`

#### 乱数・倍数
`RAND`, `RANDBETWEEN`, `GCD`, `LCM`, `FACT`

#### 条件付き集計
`IF`, `SUMIF`, `COUNTIF`, `AVERAGEIF`, `SUMIFS`, `COUNTIFS`, `AVERAGEIFS`, `IFERROR`

#### 日付・時刻
`TODAY`, `NOW`, `DATE`, `YEAR`, `MONTH`, `DAY`, `HOUR`, `MINUTE`, `SECOND`, `TIME`,
`WEEKDAY`, `WEEKNUM`, `DATEDIF`, `EDATE`, `EOMONTH`, `DAYS`

日付シリアル値は **Power BI 規約**（1899-12-30 = 0、1900 閏年バグなし）。
1900-03-01 以降は Excel と完全一致。

#### 財務
`PMT`, `PV`, `FV`, `RATE`, `NPER`, `NPV`, `IRR`

Excel と同じキャッシュフロー符号規約（受取り=正、支払い=負）。

#### 検索・参照
`VLOOKUP`, `HLOOKUP`, `INDEX`, `MATCH`

#### 文字列
`LEFT`, `RIGHT`, `MID`, `LEN`, `TRIM`, `UPPER`, `LOWER`, `CONCATENATE`/`CONCAT`

#### 論理
`AND`, `OR`, `NOT`

#### 情報
`ISBLANK`, `ISNUMBER`, `ISTEXT`

### 数式の比較は相対誤差

`=(0.1+0.2)=0.3` は **TRUE**（約 15 桁の相対誤差で等価判定）。Excel 同等の挙動。

```
=(0.1+0.2)=0.3            → TRUE
=(0.1+0.2)>=0.3           → TRUE  （境界は等価扱い）
=(0.1+0.2)>0.3            → FALSE
=COUNTIF(A1, ">=0.3")     → 1     （A1=0.1+0.2 のとき）
```

## 6. ファイル操作

### キーバインド

| キー | 動作 |
|------|------|
| `Ctrl+N` | 新規 |
| `Ctrl+O` | 開く |
| `Ctrl+S` | 保存（既存ファイル） |
| `Ctrl+Shift+S` または メニュー | 名前を付けて保存 |
| `Ctrl+P` | 印刷（HTML 出力 → ブラウザ） |

### 拡張子で自動振り分け

`開く` / `名前を付けて保存` ダイアログのファイル名は **拡張子で自動判別**：

| 拡張子 | 形式 | 備考 |
|--------|------|------|
| `.json` / なし | tbla ネイティブ | 書式・条件付き書式・シート全てを完全保存 |
| `.csv` / `.tsv` | CSV/TSV | 値のみ |
| `.xlsx` / `.xlsm` | Excel | 値・数式・列幅・書式・条件付き書式・複数シート |
| `.parquet` | Parquet | DataFrame ビューとして読み書き |

### 各形式の特徴

#### ネイティブ JSON
- 構造そのまま保存（数式・書式・条件付き書式・複数シート全部）
- Git に commit して diff レビュー可能（テキスト形式）
- 推奨用途: バージョン管理したい設計表・チェックリスト

#### CSV / TSV
- インポート: `ファイル → CSVインポート...`
- エクスポート: `ファイル → CSVエクスポート...`
- 値のみ（数式は評価結果として）
- システムクリップボード経由のコピペも TSV

#### Excel (.xlsx)
- **読込**：値・数式（再評価）、列幅、背景色・文字色・水平揃え・太字、条件付き書式（cellIs / colorScale / dataBar）
- **書込**：tbla の書式が Excel/LibreOffice で同じ見た目で開ける
- マルチシート対応
- **未対応**：罫線、斜体、下線、フォントサイズ、テーマ色（直接 RGB のみ取り込み）

#### Parquet
- 列指向圧縮形式（Snappy）。数値中心データで CSV の **約 1/10 サイズ**
- 開くと自動的に **DataFrame ビュー** で起動（高速）
- 100 万行級でも秒で開ける
- DataFrame ビューでは `名前を付けて保存` で `.parquet` 拡張子を指定すれば直接保存可能

## 7. セル書式と条件付き書式

### セル書式（手動）

選択範囲または現在セルに適用：

| 操作 | 動作 |
|------|------|
| `Ctrl+B` | 太字 ON/OFF |
| 書式 → 左/中央/右揃え | 水平揃え変更（既定は数値=右、文字=左） |
| 書式 → 文字色... | RGB 入力（`255,255,255` / `#fff` / `#ffffff` / 空でクリア） |
| 書式 → 背景色... | 同上 |
| 書式 → 数値書式... | 種別 + 小数桁数 |
| 書式 → 書式クリア | 選択範囲の書式を既定に戻す |

### 数値書式の種別

| 種別 | 例 | 出力 |
|------|-----|------|
| general | 1234.5 | `1234.5`（末尾 0 除去） |
| number 2 | 1234.5 | `1234.50` |
| currency 0 | 1234 | `¥1234`（rust_xlsxwriter 経由で書出し） |
| percent 1 | 0.15 | `15.0%` |
| scientific | 12345 | `1.23e+04` |
| date | 45292 | `2024-01-01` |
| text | 12345 | `12345`（数値を文字列扱い） |

書式は **値の編集後も保持** されます。

### 条件付き書式

セルの値に応じて自動的に色付け。`書式 → 条件付き書式...` で 3 つの入力：

1. **対象範囲**：`B2:B100`、単一セル `A1` も可
2. **条件**：
   - 比較：`>100` / `<=0` / `=42` / `<>0`
   - カラースケール：`scale:0-100` または `scale:0-100,255,255,255,220,50,50`
3. **背景色**：`255,200,200` または `#fee`

例：

```
範囲:  B2:B100
条件:  >=80
色:    #c8f0c8     → 80 以上のセルが緑系背景に

範囲:  A1:A50
条件:  scale:0-100,255,255,255,220,50,50  → 白→赤のグラデーション
色:    （空）
```

`書式 → 条件付き書式 全クリア` で全削除。

### Excel との互換性

**読込側**: tbla は xlsx 開封時に背景色・文字色・配置・太字・条件付き書式（cellIs / colorScale / dataBar）を取り込みます。

**書込側**: tbla の書式は `rust_xlsxwriter::Format` 経由で xlsx に書き出され、Excel/LibreOffice で同じ見た目で開けます。

## 8. データ操作

### 検索

| キー | 動作 |
|------|------|
| `Ctrl+F` | 検索ダイアログを開く |
| `F3` | 次を検索（前回の検索文字列を再利用） |
| `Ctrl+R` | 置換ダイアログ |
| `Ctrl+G` | セル参照を入力してジャンプ |

検索は **大小無視**、セル評価結果の部分一致。

### 置換

`Ctrl+R` で 2 フィールドダイアログ。Tab で「検索」「置換」フィールドを切替。Enter で全置換。

```
検索:  test
置換:  TEST
→ "12 件置換しました"
```

セルの **raw 入力**（数式含む）を対象に大小無視で置換します。

### 並べ替え

`データ → 並べ替え...` で 3 フィールド：

| 欄 | 入力 |
|----|------|
| 並べ替え列 | 列名（既定はカーソル列） |
| 順序 | `asc`（昇順、既定）/ `desc` |
| ヘッダー行 | `y` で 1 行目を固定（既定）/ `n` |

数値は数値、文字列は大小無視で比較。空セルは昇順で末尾。

**注意**: 数式セルが範囲内にあるとき、参照は移動後に書き換えられません。同一行内の相対参照（`=A1+B1`）は移動後も正しいまま動きますが、絶対行参照は壊れる可能性があります。

### フィルター

`データ → フィルター...` で 3 フィールド：

| 欄 | 入力 |
|----|------|
| 対象列 | 列名 |
| 条件 | `>100` / `<=0` / `=fruit` / `*example*` |
| ヘッダー行 | `y` / `n` |

条件記法：

| 記法 | 意味 |
|------|------|
| `>10` / `<10` / `>=0` / `<=100` | 数値比較 |
| `=fruit` / `<>0` | 等値・不等 |
| `*example*` | 文字列の部分一致（前後の `*` 必須） |

該当しない行を **非表示** にします（削除はしない）。`データ → フィルター解除` で全行を再表示。

**ファイル保存時にフィルターは自動解除** されます（フィルター状態はセッション限定）。

## 9. マルチシート

### シート操作

| 操作 | 動作 |
|------|------|
| `Ctrl+PgDn` / `Ctrl+PgUp` | 次 / 前のシート |
| 画面下のタブをクリック | そのシートへ |
| `シート → 新規シート` | アクティブシートの直後に追加 |
| `シート → シート名変更...` | 現在のシート名 |
| `シート → シート削除` | 現在のシートを削除（最後の 1 枚は削除不可） |

タブバーは **2 シート以上** あるときに画面下に表示されます。

### クロスシート参照

数式から他シートのセルを参照できます：

```
=Sheet2!A1                # Sheet2 の A1
=SUM(Sheet2!A1:A10)       # Sheet2 の範囲を集計
='Sales 2024'!B5          # 名前にスペースがあれば単引用符
```

**制約**：参照先のセルがさらに別シートを参照する**二段クロス参照は未対応**（`#REF!`）。同一シート内の数式は深くネスト可能。

### ファイル形式とシート

| 形式 | マルチシート保存 |
|------|---------------|
| JSON (ネイティブ) | ✅ v2 形式で完全保存 |
| Excel xlsx | ✅ 全シート読み書き |
| CSV | ❌ アクティブシートのみ |
| Parquet | ❌ アクティブシート（の DataFrame）のみ |

## 10. DataFrame ビュー

「**大量データの分析モード**」。Polars の DataFrame バックエンドで列指向・型付きデータを扱います。

### いつ使うか

| 使うべき | 使うべきでない |
|---------|---------------|
| 数万〜数百万行のデータ | 小さな手入力シート |
| 規則正しい表（1 行目=ヘッダー、列ごとに型が統一） | 行ごとに構造が違うアドホック表 |
| 集計・グループ集計・SQL 抽出が主 | セル数式を多用 |
| Parquet / 巨大 CSV を扱う | xlsx の書式編集が主 |

### 変換

```
データ → DataFrame ビューに変換
```

- 1 行目をヘッダーとして認識
- 各列の型を自動推論（Int64 / Float64 / Boolean / Utf8）
- 空セルは null

ステータスバーに `DF 1000×8 [Int64, Utf8, Float64, ...]` のように行数・列数・型サマリーが表示されます。

戻すときは `データ → セルビューに戻す`。元のセル内容は保持されているので **無損失**。

### 編集

- **行 0 のセル編集** → 列名変更（空・重複は拒否）
- **データ行のセル編集** → 値を更新。型が一致すれば保持、パース失敗時は列全体が **Utf8 に自動拡張**
- **空入力** → null
- Undo (`Ctrl+Z`) 対応

### 計算列

`データ → 計算列を追加...` で派生列を追加：

| 列名 | 式 | 結果 |
|------|-----|------|
| `revenue` | `price * qty` | 算術 |
| `tax` | `revenue * 0.1` | 既に追加した計算列を参照可能 |
| `grade` | `CASE WHEN score >= 80 THEN 'A' ELSE 'B' END` | 条件分岐 |
| `total` | `ROUND(price * 1.08, 0)` | 関数 |
| `email_ok` | `email LIKE '%@%.%'` | パターンマッチ |

式は **Polars SQL** で評価されるので、`CASE WHEN`・四則演算・組み込み関数（`ROUND`, `COALESCE`, `LENGTH` 等）が使えます。

`データ → 計算列をクリア` で全削除。

### SQL クエリ

`データ → SQL クエリ...` で **任意の Polars SQL** を実行：

```sql
SELECT * FROM df WHERE price > 100

SELECT category, SUM(amount) AS total, COUNT(*) AS n
FROM df GROUP BY category ORDER BY total DESC

SELECT * EXCLUDE name FROM df

SELECT name,
       score,
       CASE WHEN score >= 80 THEN 'pass' ELSE 'fail' END AS grade
FROM df
WHERE score > 0
ORDER BY score DESC
LIMIT 100
```

結果が DataFrame ビューを置き換え。`Ctrl+Z` で元に戻せます。

### グループ集計

SQL を書きたくない時：`データ → グループ集計...`

| 欄 | 入力 |
|----|------|
| グループ列 | カンマ区切り、例: `category, region` |
| 集計 | `col:func`、例: `amount:sum, score:avg, qty:count` |

集計関数: `sum / avg / min / max / count / stddev / var`

内部で構築される SQL（例）：

```sql
SELECT "category", "region",
       SUM("amount") AS "amount_sum",
       AVG("score") AS "score_avg"
FROM df
GROUP BY "category", "region"
```

### 直接 I/O

#### Parquet

`.parquet` ファイルは `ファイル → 開く` で **直接 DataFrame ビューとして開けます**。
書き出しは `ファイル → Parquet として保存...`（または通常の `名前を付けて保存` で `.parquet` 拡張子）。

```bash
# CLI 引数でも直接開ける
./target/release/tbla data.parquet
```

100 万行の Parquet が秒で開きます。

#### CSV を高速読込

`ファイル → CSV を DataFrame として開く...` で Polars の高速 CSV リーダーを使用。
10 MB / 数百万行の CSV も瞬時に開きます。

通常の `CSVインポート` はセル単位の読込なので大きいファイルには不向き。
DataFrame モードを使ってください。

## 11. 印刷と共有

### HTML 印刷

`Ctrl+P` または `ファイル → 印刷 (HTML)...`

- 自動的にブラウザが開く（macOS: `open`、Linux: `xdg-open`、Windows: `start`）
- ブラウザの **Cmd/Ctrl+P** で印刷ダイアログ → 余白・ページ番号・PDF 保存・縮小印刷など
- 列ヘッダー（A, B, C...）と行番号は **各ページで自動的に繰り返し** 表示

| 目的 | 推奨形式 |

## 12. ターミナル別の注意点

tbla は WezTerm / iTerm2 / Alacritty / kitty で問題なく動きます。macOS Terminal.app には既知の制限が **2 つ** あります。

### macOS — Home/End キーがない場合

Magic Keyboard など Home/End/PgUp/PgDn が無い環境では `Fn+矢印` を使用：

| 押し方 | 送られるキー |
|--------|------------|
| `Fn+←` | Home |
| `Fn+→` | End |
| `Fn+↑` | PgUp |
| `Fn+↓` | PgDn |
| `Fn+Ctrl+←` | Ctrl+Home |
| `Fn+Ctrl+→` | Ctrl+End |

### macOS Terminal.app — `Shift+↑/↓` の SHIFT が落ちる

左右は問題なし。上下方向の範囲選択が効きません。次のいずれかで対応：

1. **代替キー**: `Ctrl+Shift+K`（上）/ `Ctrl+Shift+J`（下）
2. **Terminal.app 側で修正**: 設定 → プロファイル → キーボードタブ → `+` 追加
   - キー `↑` / 修飾子 `Shift` / アクション `テキストを送信...` / 値 `\033[1;2A`
   - キー `↓` / 修飾子 `Shift` / アクション `テキストを送信...` / 値 `\033[1;2B`
3. **別のターミナルを使う**

### macOS Terminal.app — マウスのクリック・ドラッグが渡らない

Terminal.app がテキスト選択用にマウスボタンイベントを横取りするため、列幅変更のドラッグなどが動きません。**⌥ (Option) を押しながらドラッグ** すればプログラムに渡ります。

### macOS — `F1`〜`F12` がメディアキー扱い

F2 編集が動かない場合：

- **`Fn+F2`** を押す
- または システム設定 → キーボード → 「F1, F2 などのキーを標準のファンクションキーとして使用」を ON
- または セルをダブルクリックで編集開始

## 13. ワークフロー例

### A. 月次売上レポート（小〜中規模）

```bash
# 1. 新規シート起動
tbla report.xlsx

# 2. データ入力（Tab で右、Enter で下）
   A1: 日付   B1: 商品   C1: 数量   D1: 単価   E1: 売上
   A2: 2024-01-01  B2: 林檎  C2: 100  D2: 150  E2: =C2*D2

# 3. E2 をコピー、E3:E100 へ貼付（数式が自動で参照調整される）

# 4. 集計
   G1: 合計売上     H1: =SUM(E2:E100)
   G2: 平均単価     H2: =AVERAGE(D2:D100)
   G3: 商品数       H3: =COUNTA(B2:B100)

# 5. 書式
   H1〜H3 を選択 → 書式 → 数値書式... → currency 0

# 6. 条件付き書式（売上が 10000 未満は赤）
   書式 → 条件付き書式...
   範囲: E2:E100
   条件: <10000
   色:   #ffcccc

# 7. 保存
   Ctrl+S → report.xlsx
   または Ctrl+P で印刷
```

### B. 巨大 CSV のサマリー作成

```bash
# 100MB の access.log を CSV にしたもの (1000万行)
tbla

# 1. ファイル → CSV を DataFrame として開く... → access.csv
#    数秒で読込、型を自動推論

# 2. データ → SQL クエリ...
   SELECT status, COUNT(*) AS n
   FROM df
   GROUP BY status
   ORDER BY n DESC

# → ステータス別ヒット数のサマリー

# 3. データ → 計算列を追加...
   列名: hour
   式:   SUBSTR(timestamp, 12, 2)

# 4. データ → グループ集計...
   グループ列: hour
   集計:       n:count

# 5. ファイル → Parquet として保存... summary.parquet
#    次回からは tbla summary.parquet で瞬時に開ける
```

### C. Excel ファイルの加工

```bash
# Excel ユーザーから来た売上データを加工
tbla sales.xlsx

# 1. 既存のシート構造を確認
# 2. Ctrl+R で「2023年」→「2024年」一括置換
# 3. データ → 並べ替え... で日付昇順に
# 4. 条件付き書式で異常値（前年比 50% 未満）を黄色に
# 5. ファイル → 名前を付けて保存... → sales_2024.xlsx
#    （元の書式・条件付き書式も保持されて Excel で開ける）
```

### D. データクレンジング（DataFrame 編集）

```bash
tbla messy.csv

# 1. データ → DataFrame ビューに変換
# 2. 列名が "Col1" "Col2" になっていたら、F2 で各セル選択 → 入力
#    例: row 0 / col 0 で F2 → "customer_id" Enter
# 3. データ → SQL クエリ...
   SELECT customer_id, TRIM(name) AS name, amount
   FROM df
   WHERE amount IS NOT NULL AND amount > 0

# 4. 必要なら calculated column 追加
   データ → 計算列を追加...
   列名: amount_jpy
   式:   ROUND(amount * 150, 0)

# 5. ファイル → Parquet として保存... cleaned.parquet
```

## 14. トラブルシューティング

### tbla が起動しない / `error: linker ...`

→ `cargo build --release` を実行し直す。初回ビルドは Polars のコンパイルで 2〜3 分かかる。

### 数式が `#NAME?` を返す

→ 関数名のスペルミス、または tbla が対応していない関数。第 5 章の関数一覧で確認。

### `=A1+B1` が文字列扱いされる

→ `=` の前にスペースが入っていないか確認。`set_cell` は `=` で始まる入力のみ数式と判定。

### 数値書式を設定したのに反映されない

→ 書式は「セルがその時点で持っている書式」を適用。書式設定後に値を上書きすると **書式は保持** されるが、別のセルからコピペすると書式も上書きされる場合あり。

### ファイルが開けない / `os error 123` (Windows)

→ パスに `"…"` の引用符が付いていないか確認。0.2.2 で対応済みだが古いビルドの可能性。

### マウスドラッグで列幅が変わらない（macOS Terminal.app）

→ ⌥ (Option) を押しながらドラッグ。第 12 章参照。

### Shift+↑/↓ で範囲選択できない（macOS Terminal.app）

→ Terminal.app の仕様。`Ctrl+Shift+K/J` を使うか、Terminal.app の設定でキーバインドを追加。第 12 章参照。

### F2 で編集モードに入れない（macOS）

→ メディアキーの設定が干渉。`Fn+F2` または ダブルクリックで代替。

### 巨大 CSV を開くと遅い

→ `ファイル → CSV を DataFrame として開く...` を使う（通常の CSV インポートはセル単位なので遅い）。

### 条件付き書式が xlsx で消える

→ Excel/LibreOffice 側で再度開いたとき書式が正しく表示されなければ、それは tbla の書き出しに問題がある可能性。issue 報告を。

### DataFrame ビューでセル編集すると列の型が変わる

→ 期待動作。新しい値が列の型でパースできない場合、その列全体が Utf8 に自動拡張される。元の型に戻したい場合は `Ctrl+Z` で Undo。

### Undo が効かない

→ 100 段までの履歴。新規ファイルを開くと履歴クリア。
